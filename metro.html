Metro Tap In/Out System

December 2022 - December 2022

DESCRIPTION

This project was completed as the Final Project for my 10 week training with Wiley Edge. This project required me to work in a team to create a Metro tap in and tap out website, using SpringBoot (Java) and Microservices Architecture. My role within this project was to oversee that all areas of the project were functioning correctly, assist with the Front End design by ensuring that the Wireframes created by another member were reflected in our website, and to test each function and ensure that everything was cohesive. This was project was to be completed within one week.

TECHNOLOGY USED / METHODOLOGIES USED

  • Java
  • SpringBoot
  • Spring Data JPA
  • MySQL
  • HTML, CSS
  • Thymeleaf
  • Microservices Architecture
  • GitHub / Version Control
  • REST API
  • IntelliJ / Visual Studio Code

This project was created so that it would allow users to login or register for an account. If they registered, they were given £25 free credit so they could start their journeys.

Once they were logged in, they would then be presented to the View Balance page which was the homepage. Here they could view their balance. Select the station they were currently at and either tap in or out. Once they tap out, the total price for their journey was calculated and the total taken off their balance.

Firstly when creating the front-end of the website, I used Visual Studio code and plain html and css to start creating the design. I had completed a few of the pages, and another group member completed the others. Using VSCode allowed me to refresh the preview and get a real-time update of the design whereas with IntelliJ as we were using Thymeleaf for the views, it would not allow a 'quick' update. After creating the desing in VSCode, I would then transfer it into our project where I would add the Thymeleaf templating code to add functionality to the webpages and so that the pages were compatiable with the project.

After creating the pages, it was my job to ensure that the controller would then take the user to the correct pages, and that the right parameters, and data were being requested and shown.

As this project was to be completed within less than a week, the project had few errors and some features were not fully implemented as a result.
