User Experience Design Module

September 2021 - December 2021

DESCRIPTION

This university module required me to create a Movie Review website using React and Flask with the main focus being the end user's experience (UX). In order to plan and create the website, I first had to create a prototype which was tested against users through a usability test, which was then altered, and then the design was created. After the website was complete, it was tested once again to pick out any further improvements and highlight the users experience.

TECHNOLOGY USED

  • HTML
  • Flask
  • Python

I used Adobe XD to create a working prototype which emulated the design, features and functions of the website. As XD has data-entry limitations, I was limited to a partial implementation which used mock data input meaning that the prototype could not be as realistic as intended which was challending to do, but also quite fun. I enjoyed having to find work arounds to create the elements, and to learn different techniques in order to create the mock functionality.

The usability tests were conducted using Microsoft Teams to record the users as they completed set tasks. This was used to collect data such as Task Completion Times, to note any Verbal Utterances the user provided and to see any struggles the user had with the design. At the end of testing, users were asked to fill a form which provided their opinions on each of the tasks and allowed them to provide further details on features they liked/disliked.

After the formative user testing, the improvements to the design were noted and were implemented in the next stage which was the creation of the website. For the implementation of the website, I used a Flask backend using Python to handle the data for the website. To pull data from the database, I used SQLAlchemy which allowed communication with the Python and SQL server. For the UI and front-end of the website I used React which was a framework I had never used prior to this project. However, I understood and picked up layout of React very quickly which was essential in getting the project ready in time for testing and the module deadline.

Further testing was conducted with the same users after the prototype design was implemented into the website. This provided further information about how to user's experience with the website is and to see if it has been improved compared to the previous testing. Overall, the users were all very satistfied, with 85% of users saying the website was very easy to use, meaning there was an improvement of around 30% (previously 57% of users found it very easy to use).
